SCV GT

Cameron Moy

Soft contract verification for gradually-typed programs.

1 Command Line

The raco scv-gt command allows you to optimize Typed Racket programs using soft contract verification. It does so by extracting the contracts generated by Typed Racket, passing them to scv for verification, and eliminating all contracts that are proven safe statically.

The raco scv-gt command accepts the following command-line flags:

  • -s or --show-contract — print the target modules with their contracts attached.

  • -p or --provide-off — do not attach contracts to provide forms.

  • -r or --require-off — do not attach contracts to require forms.

  • -i or --ignore-check — turn all invocations of require-typed/check into require-typed.

  • -o or --overwrite — overwrite original source files with the a version with optimized contracts.

  • -c or --compiler-off — don’t compile the targets into zo files.

  • -v or --verify-off — don’t pass the modules through SCV.

If you want to optimize many files at once, pass them all as command-line arguments.
